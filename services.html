# Top Hat Cleaners — GitHub Edition Simplified website that runs as **pure HTML + CSS** for the public pages, with minimal JavaScript only where Supabase is required (booking submission and admin dashboard). Designed to live in a GitHub repository so you can edit and update from any browser. --- ## 📁 What's in This Folder ``` website-github/ ├── img/ │ ├── logo.png (cream logo for light backgrounds) │ ├── logo-dark.png (transparent map logo for dark backgrounds) │ ├── bathroom-before.jpg (your real before/after photos) │ ├── bathroom-after.jpg │ ├── room-before.jpg │ └── room-after.jpg ├── index.html (homepage — pure HTML, no JS) ├── services.html (service menu — pure HTML, no JS) ├── book.html (booking form — ~30 lines of JS for Supabase) ├── admin.html (your dashboard — needs Supabase auth) ├── styles.css (all shared styles) ├── config.js (your Supabase credentials) ├── supabase-setup.sql (run this once in Supabase) └── README.md (this file) ``` **Why some JS?** You can't talk to a database without it — the booking form needs to submit data to Supabase, and the admin needs to log in and read it. But everything visible on the public homepage and services page is 100% HTML and CSS. --- ## 🚀 Setup — Three Phases ### PHASE 1: Set Up Supabase (5 min — you said you already have this) If your Supabase project isn't fully set up yet: 1. Go to **SQL Editor** → **New Query** 2. Paste the contents of `supabase-setup.sql` → Click **Run** 3. Go to **Authentication → Users → Add user** 4. Email: `tophatcleaners210@gmail.com` · Password: choose strong one · Toggle **Auto Confirm User: ON** 5. Go to **Settings → API** and copy: - **Project URL** (e.g., `https://abcdefgh.supabase.co`) - **anon public** key (long string starting with `eyJ...`) ### PHASE 2: Push to GitHub (10 min) #### Option A — Using GitHub Web (Easiest, No Tools Needed) 1. Go to https://github.com → Sign in (create free account if needed) 2. Click the **+** in the top-right → **New repository** 3. Repository name: `tophatcleaners` (or whatever you like) 4. Set to **Public** 5. ☐ Don't initialize with README (you have your own) 6. Click **Create repository** 7. On the next page, click **uploading an existing file** 8. Drag ALL the files from your `website-github` folder into the upload area 9. **Important:** Drag the `img` folder too — make sure it goes in as a folder 10. Scroll down and click **Commit changes** #### Option B — Using GitHub Desktop (If You'll Update Often) 1. Download from https://desktop.github.com 2. File → Add Local Repository → choose your `website-github` folder 3. Publish to GitHub.com ### PHASE 3: Configure & Deploy (10 min) #### Edit config.js Through GitHub Web UI 1. In your GitHub repo, click on `config.js` 2. Click the **pencil icon** (Edit this file) at the top right 3. Replace `YOUR_SUPABASE_URL_HERE` with your Project URL (keep the quotes) 4. Replace `YOUR_SUPABASE_ANON_KEY_HERE` with your anon key (keep the quotes) 5. Scroll down and click **Commit changes** #### Enable GitHub Pages (Free Hosting) 1. In your repo, click **Settings** (top right) 2. Click **Pages** in the left sidebar 3. Under **Source**, select branch: **main** and folder: **/(root)** 4. Click **Save** 5. Wait 1-2 minutes 6. Refresh — you'll see your URL like: **https://yourusername.github.io/tophatcleaners** That's your live website! 🎩 --- ## ✏️ How to Update Anything (No Code Editor Needed) The whole point of using GitHub: you can edit any file directly in the browser. ### To change text, prices, services, etc.: 1. Check that there's an `img` folder visible in your GitHub repo with all 6 image files inside. If not, re-upload. **Booking form says "Booking system not configured":** Your `config.js` still has the placeholder values. Go back to GitHub, edit `config.js`, paste your real Supabase URL and key, commit. **I can't log into admin.html — "invalid email or password":** Three possible causes: 1. You forgot to toggle "Auto Confirm User" ON when creating the admin user. Go to Supabase → Authentication → click your user → confirm email manually. 2. Wrong password. Reset it in Supabase. 3. The admin user doesn't exist yet. Go to Supabase → Authentication → Users to check. **GitHub Pages says "Site not found":** GitHub Pages can take up to 10 minutes on first deploy. Wait, then refresh. If still broken, check **Settings → Pages** to make sure the source branch is set correctly. **I edited a file but the live site didn't update:** GitHub Pages takes 1-2 minutes to rebuild after a commit. Hard-refresh your browser (Ctrl+Shift+R or Cmd+Shift+R). If still showing old content after 5 minutes, check the Actions tab in GitHub for any deploy errors. **I want to use a custom domain (tophatcleaners.com):** Once you have your first paying client, buy a domain from Namecheap or Google Domains ($12/year). In GitHub: Settings → Pages → Custom domain. Follow GitHub's DNS instructions. --- ## 📊 Daily Habits After Going Live - **Daily:** Check `admin.html` for new bookings - **Daily:** Respond to inquiries within 30 minutes during business hours - **Weekly:** Add a new before/after photo set (huge for Google rankings + social proof) - **Monthly:** Ask happy clients for Google reviews — these dramatically improve local search rankings --- 🎩 **Top Hat Cleaners — Cleaning with Class** San Antonio, TX · (726) 888-9821 · tophatcleaners210@gmail.com
